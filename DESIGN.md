## Identitet

**Kontrasten är konceptet.**  
Systemgränssnittet är rent, tyst och precist — som Apple. In i det systemet placeras grova, bläckiga, handgjorda manicula-händer. Mötet mellan de två är vad som gör Kantarell igenkännbart.

- Modern UI-sida: nästan vitt, systemtypsnitt, tunna strokes, generöst luft
- Vintage-sida: Playfair Display i kursiv, manicula-händer, orange accent
- Orange används *sparsamt* — det ska synas, inte dominera
- Aldrig pompöst. Aldrig välkomsttexter. Aldrig "fantastisk upplevelse".

## Färgpalett

| Namn | Hex | Användning |
|------|-----|------------|
| Bläck | `#111111` | Primär text, knappar, konturer |
| Is | `#F7F7F5` | Sidans bakgrund (kall, inte varm) |
| Vitt | `#FFFFFF` | Kortbakgrunder, input-fält |
| Granit | `#666662` | Sekundär text, metadata, etiketter |
| Dimma | `#E2E2DF` | Kantlinjer, avgränsare |
| **Kantarell** | `#D96B1F` | **Accentfärg — den enda färgen** |
| Orange tone | `#F5DFC0` | Callout-bakgrunder |
| Orange yta | `rgba(217,107,31,0.06)` | Subtila markeringsytor |

**Nyckelprincip:** Bakgrunden är kall och neutral. Det gör att oranget syns tydligare med mindre ansträngning — precis som Apple hanterar sin blå accent.

**Mörkt läge:**
```css
/* Dark mode */
--bg:    #111111;
--bg2:   #1A1A1A;
--w:     #1F1F1F;
--ink:   #F0F0EE;
--mid:   #888884;
--faint: #2A2A28;
/* Orange och Kantarell behålls oförändrade */
```

## Typografi

Två tydliga system — blanda dem inte.

### System / UI-typsnitt
```css
font-family: -apple-system, BlinkMacSystemFont, 'Helvetica Neue', Arial, sans-serif;
```
Används för: knappar, etiketter, input-fält, metadata, navigationselement, all funktionell text.  
Laddar instantant — inga externa resurser för UI.

### Display / Rubrik
**Playfair Display** (Google Fonts, gratis)  
```css
font-family: 'Playfair Display', Georgia, serif;
```
Används för: h1–h2, logotyptyp, stora citat, sidrubriker.  
Använd `font-style: italic` och `font-weight: 400` för maximal karaktär.  
Undvik bold i rubriker — Playfair är starkast i regular kursiv.

```html
<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Playfair+Display:ital,wght@0,400;0,600;1,400&display=swap" rel="stylesheet">
```

### 2. Vintage manicula — för karaktär och "titta här"-moment

Manicula används **aldrig** för navigation eller funktion.  
Manicula används för: varningar, tips, viktiga callouts, "notera detta", signaturer.

#### Källor (prioritetsordning)

**Adobe Stock** *(ingår i Adobe CC — bästa kvaliteten)*  
Sök: `"Victorian manicule vector"` eller `"pointing hand engraving"` eller `"vintage woodcut pointing finger"`  
Format: AI/EPS → öppna direkt i Illustrator → exportera som SVG.  
Ingen extra kostnad med ditt CC-abonnemang.

**The Graphics Fairy** — `thegraphicsfairy.com`  
11+ viktorianska manicula från 1820–1910-talet. Gratis för kommersiellt bruk (bekräftat i FAQ).  
Flöde: Ladda ner PNG → Illustrator Image Trace (preset: "Silhouette") → Expandera → Rensa → SVG.

**rawpixel.com**  
Professionellt restaurerade public domain-manicula. Sök `"manicule"` eller `"pointing hand vintage"`.  
Format: PNG och SVG. CC0 — ingen attribution krävs.

**publicdomainvectors.org**  
2000+ viktorianska pekhänder som redigerbar AI/EPS/SVG. Öppna direkt i Illustrator.

**Wikimedia Commons**  
Originalinscanningar 300–600 dpi. Sök: `manicule` eller `typographic pointing hand`.  
Bästa underlaget för Image Trace i Illustrator.

#### Illustrator-flöde för egna manicula
```
1. Hämta hög-dpi PNG (minst 600px bred)
2. Placera i Illustrator
3. Image Trace → preset "Silhouette" eller "Black and White Logo"
4. Tröskel: 128 (justera efter bild)
5. Expandera → Ungroup
6. Ta bort bakgrundsrektangel
7. Rensa ankarpunkter (Object → Path → Simplify, 95%)
8. Exportera som SVG (Presentation Attributes)
9. Optimera med SVGO eller svgomg.net
```

## Röst och ton

### Fem principer
1. **Kortfattat** — säg det på halva orden
2. **Konkret** — siffror, platser, tider, inte vaga påståenden
3. **Lite fyndigt** — ett lättsamt ordval per sida räcker, inte fler
4. **Aldrig pompöst** — inga välkomsttexter, inga "fantastiska upplevelser"
5. **Svenska i första hand** — engelska bara för etablerade tekniska termer

### Tonexempel

| Situation | Kantarell-röst | Undvik |
|-----------|---------------|--------|
| Tom lista | `Inga resultat. Prova ett bredare område.` | `Tyvärr hittades inga resultat för din sökning.` |
| Laddar | `Hämtar data…` | `Vänligen vänta medan vi laddar din information.` |
| Fel | `Kunde inte ansluta. Försök igen.` | `Ett oväntat fel inträffade. Vänligen kontakta support.` |
| Sparat | `Sparat.` | `Din information har sparats framgångsrikt!` |
| Hjälptext | `Klicka för detaljer.` | `Klicka på valfri punkt i diagrammet för att visa detaljerad information.` |
| Inget valt | `Välj ett område för att börja.` | `Välkommen! Använd kartan nedan för att utforska data.` |

### Tomma tillstånd (empty states)
Använd alltid: liten manicula-ikon + en mening + valfri åtgärdsknapp.  
Aldrig bara tom yta.

## Manicula — användningsregler

| Använd | Använd inte |
|--------|-------------|
| Callouts och tips | Navigation |
| Viktiga varningar | Dekorativ fyllnad |
| "Notera detta"-moment | Listor med mer än 3 punkter |
| Signaturer och README-rubriker | Ersättning för systemikoner |
| En per sida/sektion max | Tillsammans med annan accentfärg |
